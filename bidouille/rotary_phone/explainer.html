Dialing the past into the present

A rotary phone, a Raspberry Pi, and two numbers — 01 plays a sound, 02 opens a stream. Here's how the wires, pulses, and code conspire.

~7 min read Hardware + Python Rev. 2026-05-10

A rotary phone doesn't speak USB. It speaks pulses. When you spin the dial to 3 and let it return, the receiver clicks the line three times — open, close, open, close, open, close. That tiny rhythm is all you need. The Raspberry Pi listens to those clicks on a GPIO pin, counts them, and decides what to do. Dial 01, hear a recorded voice. Dial 02, fall into a live audio stream. The phone becomes a remote for invisible things.

Mental model The phone is a switch that blinks Morse code in your hand. The Pi is a patient ear: it counts blinks per dial, and once you hang up the receiver, the number is "submitted" — like pressing Enter on a tiny mechanical keypad with one key.

01 · The two signals you need

An old rotary phone (think French Socotel S63 or US Western Electric 500) hides two switches that matter:

  1. The hook switch — closed when the receiver is on the cradle, open when you pick it up. This is your "start / hang up" signal.
  2. The dial pulse switch — opens and closes once per digit when the dial returns to rest. Dial 5 → 5 pulses. Dial 0 → 10 pulses (zero is treated as ten).

That's it. Two switches, two GPIO pins, and the rest is software.

Why pulses are messy Mechanical contacts bounce. A single click can register as 3-4 micro-pulses for a few milliseconds. So we don't count raw edges — we debounce: ignore any edge that arrives less than ~10 ms after the last one. Without debouncing, dialing "1" might look like "5".

Both inputs use the Pi's internal pull-up resistors: the line idles HIGH, and each pulse pulls it LOW. No external resistors needed for a simple build.

04 · The router: 01 vs 02

Dial

01

Plays a local audio file — say, welcome.mp3. Good for: greeting, voicemail, a poem. The Pi just calls mpg123 welcome.mp3.

Dial

02

Opens a live HTTP audio stream — a webradio, an Icecast feed, or your own URL. The Pi runs mpv https://stream.example/live.mp3 until you hang up.

05 · The Python sketch

# phone.py — rotary router on a Raspberry Pi (thread-safe)
import subprocess, time, threading
from gpiozero import Button

DIAL = Button(17, pull_up=True, bounce_time=0.020)
HOOK = Button(27, pull_up=True, bounce_time=0.080)

lock = threading.Lock()      # on_pulse runs on a callback thread
pulses, digits, last_pulse, player = 0, [], 0.0, None

def on_pulse():
    global pulses, last_pulse
    with lock:
        pulses += 1
        last_pulse = time.time()

def reset_state():
    # lift OR hang-up: clean slate, kill any running player
    global pulses, player
    with lock:
        pulses = 0
        digits.clear()
    if player and player.poll() is None:
        player.terminate()
        try: player.wait(timeout=1)
        except subprocess.TimeoutExpired: player.kill()
    player = None

def route(num):
    global player
    cmd = {"01": ["mpg123", "-q", "welcome.mp3"],
           "02": ["mpv", "--no-video", "https://stream.example/live.mp3"]}.get(num)
    if not cmd: return
    try:    player = subprocess.Popen(cmd)
    except FileNotFoundError as e: print(f"⚠ {e}")

DIAL.when_pressed  = on_pulse
HOOK.when_released = reset_state   # lift  → start fresh
HOOK.when_pressed  = reset_state   # cradle → stop & clean

while True:
    # 1. snapshot pulses if the dial has gone silent (digit complete)
    with lock:
        idle = pulses and (time.time() - last_pulse) > 0.25
        snap = pulses if idle else 0
        if idle: pulses = 0
    if snap:
        digits.append(0 if snap == 10 else snap)
        if len(digits) == 2:
            route("".join(map(str, digits)))
            digits.clear()
    # 2. invalidate a partial dial that's gone stale (>5 s with no new pulse)
    if digits and time.time() - last_pulse > 5.0:
        digits.clear()
    time.sleep(0.02)

Four small ideas hold this up: count edges under a lock, flush after silence, reset on hook events, drop stale partial dials. Everything else is decoration.

06 · Things that will surprise you

  1. Old phones use 48 V ringing voltage. You don't need it for input — just the contacts. But never wire the phone to a live PSTN line and to the Pi at the same time.
  2. Pulse rate varies by country. Most phones pulse at 10 Hz, but some UK phones do 20 Hz. Tune your debounce window to taste.
  3. The earpiece is a real speaker. Cut the receiver cord, solder it to a 3.5 mm jack, and the Pi can play through the handset itself. That's when the magic clicks.
  4. Latency is part of the charm. Don't fight the dial's slowness. The 6-second journey from "lift receiver" to "audio plays" is what makes the object feel alive.
